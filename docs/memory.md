# 记忆设计

第一版用 SQLite，数据默认在 `brain/data/pet_memory.db`。

数据库结构由 `brain/migrations/` 下的 SQL 迁移文件管理。首次启动前可以手动执行：

```bash
make migrate-brain-db
```

## 记忆类型

- **显式记忆**：用户说“记住……”，直接保存。
- **隐式记忆**：LLM 从对话中提炼长期有用信息。
- **情节记忆**：每轮对话保存一条 episode，方便后续回看和总结。

## 为什么先用 SQLite

- 本地可控，适合早期原型。
- 容易备份和迁移。
- 不依赖额外服务，硬件调试时少一个变量。

## 升级方向

当对话量变多后，可以把 `MemoryStore` 替换为：

- mem0：适合按用户、会话、Agent 检索长期记忆。
- Letta：适合做更完整的有状态 Agent。
- 向量数据库：适合语义检索，但第一版不是必需。
